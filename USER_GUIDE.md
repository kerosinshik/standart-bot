# Руководство пользователя - RAG система по нормативным документам

## Что умеет система

Система позволяет задавать вопросы по нормативным документам и получать точные ответы с указанием источников.

### Документы в базе знаний

1. **N 76** - Требования к лекарственным средствам
2. **N 77** - Правила надлежащей производственной практики (GMP)
3. **N 78** - Правила регистрации и экспертизы лекарственных средств
4. **N 79** - Правила надлежащей клинической практики (GCP)
5. **N 80** - Правила надлежащей дистрибьюторской практики (GDP)
6. **N 81** - Правила надлежащей лабораторной практики (GLP)
7. **N 82** - Общие требования к лекарственным средствам

**Всего:** 17,120 фрагментов текста из 7 документов

---

## Как пользоваться

### 1. Базовый режим работы

Просто введите вопрос в текстовое поле и нажмите "Отправить".

**Примеры вопросов:**

- "Что такое фармацевтическая система качества?"
- "Как проводится обзор качества продукции?"
- "Кто входит в ключевой персонал?"
- "Какие требования к валидации процессов?"
- "Что такое аудит клинического исследования?"

### 2. Режимы поиска

Система автоматически выбирает оптимальный режим поиска:

- **🔍 Векторный поиск** (по умолчанию) - для сложных вопросов, поиска по смыслу
- **⚡ Ключевые слова** - для точных запросов (коды, определения с "что такое")
- **🤖 Авто** - система сама решает

### 3. Дополнительные возможности

#### История диалога
Система помнит контекст последних 2-3 сообщений. Можно задавать уточняющие вопросы:

```
Вы: Что такое валидация?
Бот: [ответ про валидацию]
Вы: А какие виды валидации бывают?
Бот: [ответ с учетом контекста]
```

#### Выбор AI модели
- **Z.ai (glm-4.6)** - по умолчанию, быстрая китайская модель
- **Claude 3.5 Sonnet** - для более сложных вопросов (если настроена)

#### Режимы знаний
- **Строгий режим** - только информация из документов
- **Расширенный режим** - может дополнить общими знаниями AI

## Интерпретация ответов

### Структура ответа

```
**Краткий ответ**

Детали:
- Пункт 1
- Пункт 2

**Источники:** [Документ N_77, стр. 12]
```

### Метрики запроса

В конце каждого ответа отображаются:
- **AI Модель** - какая модель использовалась
- **Режим поиска** - векторный/ключевые слова/авто
- **Время поиска** - сколько заняло нахождение контекста
- **Время AI ответа** - сколько модель думала над ответом
- **Токены** - сколько токенов использовано (промпт + ответ)

### Если система не нашла ответ

Возможные причины:
1. Информации действительно нет в документах
2. Вопрос сформулирован слишком абстрактно - попробуйте переформулировать
3. Нужная информация есть, но не попала в топ результатов - попробуйте задать вопрос по-другому

## Советы по эффективному использованию

### ✅ Хорошие вопросы

- Конкретные: "Как оформляется фармацевтическая система качества?"
- С упоминанием терминов из документов: "Что входит в обязанности уполномоченного лица?"
- Про процедуры: "Как проводится валидация процессов?"

### ❌ Плохие вопросы

- Слишком общие: "Расскажи про производство"
- Не относящиеся к документам: "Как открыть фармацевтическую компанию?"
- Требующие расчетов: "Рассчитай дозировку для 70кг пациента"

### 💡 Полезные приемы

1. **Узнайте список документов:**
   ```
   Какие документы у тебя есть?
   ```

2. **Уточняйте номер документа:**
   ```
   Найди в документе N_77 информацию про ключевой персонал
   ```

3. **Задавайте уточняющие вопросы:**
   ```
   Вопрос 1: Что такое валидация?
   Вопрос 2: Приведи примеры из документа
   ```

4. **Просите больше деталей:**
   ```
   Расскажи подробнее про требования к оборудованию
   ```

## Ограничения системы

1. **Только текстовая информация** - таблицы, схемы, рисунки из PDF распознаются частично
2. **Водяные знаки** - в документах есть водяные знаки КонсультантПлюс, которые могут разрывать текст
3. **Контекст** - система анализирует ~20 фрагментов документов, длинные разделы могут быть показаны частично
4. **Актуальность** - документы на момент загрузки в систему (версии от 03.11.2016 с изменениями до 04.07.2023)

# Варианты улучшения системы

## 1. Увеличение полноты ответов

### Проблема
Ответы иногда обрываются или неполные.

### Решения

#### A. Увеличение лимита токенов (легко, рекомендуется)
- **Сейчас:** 2500 токенов (~1500-2000 слов)
- **Предложение:** 4000-5000 токенов
- **Плюсы:** Более полные ответы
- **Минусы:** Немного дороже, медленнее
- **Реализация:** Изменить `max_tokens` в `ai_client.py`

#### B. Улучшение промпта (легко, рекомендуется)
- Добавить в системный промпт требование давать ПОЛНЫЕ и ДЕТАЛЬНЫЕ ответы
- Указать, что нужно использовать ВСЮ найденную информацию
- **Плюсы:** Бесплатно, быстро
- **Минусы:** Может увеличить расход токенов
- **Реализация:** Изменить промпт в `ai_client.py`

#### C. Увеличение n_results (средне)
- **Сейчас:** 20 результатов поиска
- **Предложение:** 25-30 результатов
- **Плюсы:** Больше контекста для AI
- **Минусы:** Медленнее поиск, больше токенов
- **Реализация:** Изменить `n_results` в `main.py`

#### D. Увеличение размера чанков (сложно, требует переиндексации)
- **Сейчас:** 800 символов на чанк
- **Предложение:** 1200-1500 символов
- **Плюсы:** Меньше разрывов текста, полнее контекст
- **Минусы:** Требует полной переиндексации базы (~30 мин), больше размер базы
- **Реализация:** Изменить `chunk_size` в `init_db.py` и переиндексировать

## 2. Улучшение точности поиска

### A. Гибридный поиск (средне)
Комбинация векторного + keyword поиска с реранкером.

**Реализация:**
- Добавить BM25 для keyword поиска
- Комбинировать результаты с весами
- Использовать reranker модель для финальной сортировки

**Плюсы:** Лучшая точность, меньше пропущенных результатов
**Минусы:** Сложнее, медленнее

### B. Улучшение эмбеддингов (сложно)
- **Сейчас:** `cointegrated/rubert-tiny2` (малая модель)
- **Предложение:** `intfloat/multilingual-e5-large` или `BAAI/bge-m3`
- **Плюсы:** Лучшее качество эмбеддингов
- **Минусы:** Требует переиндексации, больше памяти, медленнее

### C. Fine-tuning эмбеддинг модели (очень сложно)
Дообучить модель на ваших конкретных документах.

**Требует:** Создание датасета вопрос-ответ, GPU, время на обучение

## 3. Улучшение обработки PDF

### A. Удаление водяных знаков (средне)
- Предобработка PDF для удаления повторяющихся паттернов (КонсультантПлюс)
- **Плюсы:** Чище текст, меньше разрывов
- **Минусы:** Требует переиндексации

### B. Извлечение таблиц (средне)
- **Сейчас:** Таблицы извлекаются частично через паттерны
- **Предложение:** Использовать `tabula-py` или `camelot` для точного извлечения таблиц
- **Плюсы:** Структурированные данные из таблиц
- **Минусы:** Требует переиндексации, сложнее обработка

### C. OCR для изображений (сложно)
Если в PDF есть сканированные страницы или важные схемы.

## 4. Дополнительные возможности

### A. Экспорт ответов в Word/PDF (легко)
Кнопка "Скачать ответ" с форматированием.

### B. Сохранение истории диалогов (средне)
- База данных с историей вопросов пользователей
- Возможность продолжить диалог позже
- **Требует:** PostgreSQL или SQLite

### C. Сравнение разных версий документов (сложно)
Показывать изменения между версиями документов.

### D. Мультиязычность (средне)
Поддержка вопросов и ответов на английском/других языках.

### E. Цитирование с прямыми ссылками (легко)
Кликабельные ссылки на страницы PDF в источниках.

## 5. Оптимизация производительности

### A. Кэширование частых запросов (средне)
- Redis для кэширования ответов на популярные вопросы
- **Плюсы:** Быстрее, дешевле
- **Минусы:** Требует Redis, память

### B. Асинхронная обработка (легко)
Уже частично реализовано через FastAPI, можно улучшить.

### C. Балансировка нагрузки (сложно)
Если много пользователей - несколько инстансов сервера.

## Техническая информация

- **Сервер:** FastAPI на Python 3.12
- **Векторная БД:** ChromaDB (персистентное хранилище)
- **Эмбеддинг модель:** cointegrated/rubert-tiny2 (CPU)
- **AI модели:** Z.ai (glm-4.6), Claude 3.5 Sonnet
- **Размер базы:** ~500 МБ (17,120 чанков)
- **Параметры:**
  - chunk_size: 800 символов
  - chunk_overlap: 800 символов (100%)
  - n_results: 20 фрагментов
  - max_tokens: 2500 (AI ответ)
